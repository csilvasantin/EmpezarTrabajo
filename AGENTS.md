# AGENTS

## 1 2 3 basico

1. ejecutar `./start_work.sh`;
2. hacer onboarding antes de tocar el repo activo;
3. cerrar con HTML y URL publica verificable.

Antes de trabajar en cualquier repo tecnico:

1. comprobar que `gh auth status` es valido;
2. ejecutar `./start_work.sh`;
3. abrir `csilvasantin/onboarding`;
4. hacer onboarding leyendo `WELCOME_PACK.md`, `CURRENT_STATE.md` y `WORKING_RULES.md`;
5. solo despues continuar con el repositorio de trabajo.

## Regla principal

No actuar como si la sesion empezara desde cero si el contexto ya existe en `onboarding`.

La clave `Nomeacuerd0` solo cuenta como filtro ligero si se usa en una pagina publica.
